# 📡 CC1101 USB↔RF Bridge: Source
*A deterministic USB‑to‑RF transport layer for CC1101‑based embedded nodes*

This repository contains a compact, production‑quality firmware module that turns an STM32‑based board with a TI CC1101 sub‑GHz transceiver into a **USB‑to‑RF packet bridge**. The design emphasizes deterministic state machines, interrupt‑safe buffering, and a minimal RF packet format suitable for text or binary streaming.

The system is intentionally simple: one node acts as a **source** (USB → RF), the other as a **sink** (RF → USB). This keeps the RF protocol minimal and easy to reason about while providing a solid foundation for more advanced protocols.

---

## 🚀 Features

- Variable‑length RF packets (1–61 bytes payload)
- Whitening + CRC enabled in hardware
- 38.4 kbps 2‑FSK at 433.92 MHz
- USB CDC streaming on the host side
- Snapshot‑based TX buffering (no IRQ races)
- Deterministic, non‑blocking state machine
- Correct CC1101 reset sequence (TI‑recommended)
- FIFO‑safe burst writes with bounded timing
- Clean, documented CC1101 driver

---

## Architecture Overview

The firmware is built around two cooperating components:

### 1. USB Accumulation Buffer
Filled by the USB CDC interrupt handler.  
Acts as a simple byte‑stream sink.

### 2. RF Transmission State Machine
Runs in the main loop.  
Consumes accumulated USB bytes and emits RF packets.

State flow:
```
IDLE → ACCUM → TX_PREP → TX_WAIT → IDLE
```

This ensures:

- No blocking inside interrupts  
- No logging inside RF timing‑critical windows  
- No races between USB IRQ and RF TX  
- Clean, deterministic packet boundaries  

---

## 📦 RF Packet Format

The RF protocol is intentionally minimal:

```
+---------+-------------------------+
| Length  | Payload (0–61 bytes)    |
+---------+-------------------------+
1 B             N bytes
```

- `Length` is the number of payload bytes (0–61)
- CRC and whitening are handled by the CC1101 hardware
- The receiver reads the first byte, then reads `Length` bytes from the FIFO

This format is ideal for:

- Text messages  
- Binary streams  
- Simple command/response protocols  
- Higher‑level framing layered on top

## 🔄 USB → RF State Machine

The TX state machine is fully documented in `RF_USB_StateMachine.cpp`.

### State Summary

#### **IDLE**
Waiting for USB data.

#### **ACCUM**
USB IRQ fills `usb_rf_buf[]`.  
When a timeout expires, snapshot the buffer.

#### **TX_PREP**
- Force CC1101 to IDLE  
- Flush TX FIFO  
- Write `[len][payload…]`  
- Issue `STX`  

No logging occurs inside the timing‑critical window.

#### **TX_WAIT**
Poll MARCSTATE until TX completes (`IDLE`).  
Optional detection of true underflow (`MARCSTATE=0x16`).
